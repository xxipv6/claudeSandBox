# Research Logging

## 何时记录

只在关键节点记录：
- 路径切换
- 关键漏洞发现
- 关键假设验证完成
- 启用多 Agent
- 形成可复现证据

普通查文件、普通搜索、简单命令操作、不产生信息增量的失败尝试，不要求记录。

---

## 最小模板

```markdown
# Step Record: step-N

- 当前目标：
- 本步动作：
- 关键发现：
- 证据位置：
- 下一步：
```

---

## 规则

- Step ID 使用 `step-N`
- 记录重点是发现、证据、下一步
- 不要求把所有中间过程写成大报告
- 需要上下文恢复时，再补充当前状态摘要
