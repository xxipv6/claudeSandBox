# Research Task Classification

## 任务复杂度分类

### 🔴 高复杂度任务

**特征**：
- 系统级审计（完整应用、固件、基础设施）
- 深度逆向（强混淆、复杂协议、状态机）
- 攻击链构建（多步骤、多组件联动）
- 不确定性高（多种可能的路径）

**示例**：
- IoT 固件完整安全审计
- 复杂协议逆向（自定义协议、状态机）
- 攻击链研究（多个漏洞组合利用）
- 大型应用源码审计
- 虚拟化/容器安全研究

**处理方式**：
1. 必须先使用 **brainstorming** 进行设计探索
2. 生成 **Decision Record**
3. 使用 **research-planner** 生成研究计划
4. 考虑使用 **Multi-Agent Strategy**

**触发流程**：
```
识别到高复杂度任务
    ↓
brainstorming（设计探索）
    ↓
用户批准设计方案
    ↓
research-planner（生成研究计划）
    ↓
Research Lead 执行研究
```

---

### 🟡 中低复杂度任务

**特征**：
- 单点漏洞分析
- 已知漏洞复现
- 明确的代码审计范围
- 单一功能模块测试

**示例**：
- 复现 CVE-2024-XXXX
- 审计单个 API 接口
- 分析特定函数的安全性
- 验证单个漏洞假设

**处理方式**：
1. 直接决策（不需要 brainstorming）
2. 生成 **Decision Record**（如需要）
3. 直接执行研究
4. 使用 **Single-Agent Strategy**

**触发流程**：
```
识别到中低复杂度任务
    ↓
直接决策研究路径
    ↓
执行研究
    ↓
每步记录
```

---

### 🟢 简单操作

**特征**：
- 信息查看
- 日志检查
- 文件操作
- 查询类任务

**示例**：
- 查看文件内容
- 搜索特定字符串
- 检查进程状态
- 查看网络连接
- 分析单个请求/响应

**处理方式**：
- 直接执行
- 不需要 Decision Record
- 不需要 Step Record
- 不需要 Specialist Agents

**触发流程**：
```
识别到简单操作
    ↓
直接执行
    ↓
返回结果
```

## 判定标准

### 高复杂度判定（满足任一即可）

- [ ] 需要审计 ≥ 3 个攻击面
- [ ] 需要逆向未知协议/格式
- [ ] 需要分析 ≥ 10 个文件
- [ ] 需要多阶段验证（静态 + 动态）
- [ ] 存在明显的研究路径分叉
- [ ] 预计耗时 > 2 小时
- [ ] 需要多 Agent 协作

### 中低复杂度判定（满足任一即可）

- [ ] 单一攻击面
- [ ] 已知漏洞类型
- [ ] 明确的验证方法
- [ ] 文件数量 < 10
- [ ] 预计耗时 < 2 小时

### 简单操作判定（满足任一即可）

- [ ] 只读操作（不修改系统）
- [ ] 查询类任务
- [ ] 单命令可完成
- [ ] 预计耗时 < 5 分钟

## 边界情况

### 情况 1：复杂度不确定

**任务**：审计这个功能模块

**你**：
1. 先快速探索（5-10 分钟）
2. 评估实际复杂度
3. 根据评估结果选择处理方式

---

### 情况 2：任务升级

**任务**：复现这个漏洞

**你**：
- 初始判断：中低复杂度
- 执行中发现：需要深入逆向协议
- 升级为：高复杂度
- 调整：启用 brainstorming + research-planner

---

### 情况 3：任务降级

**任务**：完整系统审计

**你**：
- 初始判断：高复杂度
- 探索后发现：只涉及简单配置问题
- 降级为：中低复杂度
- 调整：直接决策，执行修复

---

## 快速决策树

```
开始任务
    ↓
需要探索方案吗？
    ├─ 是 → 高复杂度 → brainstorming → research-planner → 执行
    └─ 否 ↓
        需要多步骤研究？
            ├─ 是 → 中低复杂度 → 直接决策 → 执行
            └─ 否 ↓
                简单操作？→ 是 → 直接执行
```

## 示例

### 示例 1：高复杂度

**人类**：研究这个 IoT 固件

**你**：
- 判断：高复杂度（固件解包 + 二进制分析 + 协议逆向）
- 流程：brainstorming → research-planner → Multi-Agent 研究

---

### 示例 2：中低复杂度

**人类**：复现 CVE-2024-1234

**你**：
- 判断：中低复杂度（已知漏洞，有公开 PoC 参考）
- 流程：直接决策 → 编写 PoC → 验证

---

### 示例 3：简单操作

**人类**：查看这个文件的内容

**你**：
- 判断：简单操作
- 流程：直接 cat 文件 → 返回结果

---

## 记忆要点

- **高复杂度** = 不确定性高 → 需要规划
- **中低复杂度** = 相对明确 → 直接决策
- **简单操作** = 查询类 → 直接执行

**关键**：快速判断，动态调整。
