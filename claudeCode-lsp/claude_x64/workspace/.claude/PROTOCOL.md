# Orchestrator 协议声明

## 协议性质

这是一个**人为定义的多 Agent Orchestrator 协议系统**。

### 系统定义

- **不是** Claude Code 的官方支持模式
- **不是** 自动发现的配置系统
- **不是** 配置文件即行为的系统
- **是** 人为定义的强制执行协议
- **是** 通过 CLAUDE.md 作为唯一入口的系统

### 核心原理

```
CLAUDE.md（唯一真理源）
    ↓
定义协议和规则
    ↓
Claude 必须显式读取配置文件
    ↓
按照配置执行流程
    ↓
状态文件持久化
```

---

## 强制协议（必须遵守）

### 协议 1：唯一真理源

**规则**：
- CLAUDE.md 是唯一入口
- 所有行为必须由 CLAUDE.md 定义
- 禁止偏离 CLAUDE.md 的定义

**违规后果**：
- 系统行为不可预测
- 可能跳过关键步骤
- 可能脑补或假设

---

### 协议 2：显式引用

**规则**：
- 所有文件必须显式读取
- 必须声明"现在读取 X 文件"
- 禁止自动发现或扫描目录

**正确示例**：
```
现在读取：`.claude/workflow/config.yaml`
必须解析其中的 stages 定义
```

**错误示例**：
```
.claude/ 目录下应该有配置文件
（自动扫描和发现）
```

**违规后果**：
- 文件可能不存在
- 可能读取错误的文件
- 可能跳过必要的配置

---

### 协议 3：状态驱动

**规则**：
- 状态只能来自文件
- 禁止脑补或假设状态
- 每次操作后必须更新状态文件

**状态文件位置**：
- 任务状态：`.claude/task_states/task-{id}.json`
- 子任务状态：`.claude/subtask_queues/task-{id}.json`
- 任务依赖：`.claude/task_dependencies.json`

**正确示例**：
```
当前状态来自：`.claude/task_states/task-001.json`
状态：running
当前阶段：03-mode-execution
```

**错误示例**：
```
我认为现在应该进入下一阶段
（脑补状态）
```

**违规后果**：
- 状态不一致
- 可能重复执行
- 可能跳过关键步骤

---

### 协议 4：严格顺序

**规则**：
- 按照配置的 stages 顺序执行
- 禁止跳过、合并、简化阶段
- 每个阶段必须完成才能进入下一阶段

**流程顺序**：
1. 00-planning（任务规划）
2. 01-task-init（任务初始化）
3. 02-git-prepare（Git 准备）
4. 03-mode-execution（模式执行）
5. 04-quality-gate（质量门禁）
6. 05-completion（完成与状态管理）

**违规后果**：
- 可能跳过关键步骤
- 可能导致依赖不满足
- 可能导致状态不一致

---

## 禁止行为（严格禁止）

### 行为层面

- ❌ 自动扫描或发现 .claude/ 目录
- ❌ 假设任何文件的内容
- ❌ 跳过、合并、简化任何阶段
- ❌ 使用"默认行为"替代文件内容
- ❌ 脑补状态或推断下一步
- ❌ 忽略检查点

### 思维层面

- ❌ "应该有..."
- ❌ "大概是..."
- ❌ "可以跳过..."
- ❌ "可以合并..."
- ❌ "差不多就行..."

## 正确行为示例

### 示例 1：读取配置

**正确**：
```
第 1 步：读取流程配置
现在读取：`.claude/workflow/config.yaml`
必须解析其中的 stages 定义
检查点：[ ] 已读取并解析 config.yaml
```

**错误**：
```
读取流程配置
（没有明确文件路径）
（没有检查点）
```

### 示例 2：执行阶段

**正确**：
```
阶段 1：执行阶段
对于 config.yaml 中的每个 stage：
1. 读取 stage 文件：`.claude/workflow/stages/{id}.md`
2. 按照 stage 文件中的步骤执行
3. 每步完成后更新状态文件
4. 检查点：[ ] 当前 stage 已完成
```

**错误**：
```
执行各个阶段
（没有明确文件路径）
（没有状态更新）
（没有检查点）
```

### 示例 3：状态管理

**正确**：
```
当前状态来自：`.claude/task_states/task-001.json`
读取状态：{"status": "running", "current_stage": "03-mode-execution"}
更新状态：{"status": "running", "current_stage": "04-quality-gate"}
写入：`.claude/task_states/task-001.json`
```

**错误**：
```
现在应该进入质量门禁阶段
（没有读取状态）
（没有更新状态）
```

## 检查点机制

每个阶段必须有检查点：

**检查点格式**：
```
检查点：
- [ ] 已读取配置文件
- [ ] 已执行步骤 1
- [ ] 已执行步骤 2
- [ ] 已更新状态文件
```

**检查点作用**：
- 确保每步都完成
- 防止跳过或遗漏
- 提供执行追踪

---

## 错误处理

### 文件不存在

**正确处理**：
```
错误：文件不存在：`.claude/workflow/config.yaml`
必须：明确报错
禁止：继续执行或使用默认行为
必须：停止并等待人工介入
```

### 格式错误

**正确处理**：
```
错误：配置文件格式错误
必须：明确报错
禁止：推断或假设正确格式
必须：停止并等待人工介入
```

### 超时

**正确处理**：
```
错误：操作超时（120s）
必须：明确报错
禁止：无限等待或跳过
必须：停止并等待人工介入
```

## 系统架构

### 目录结构

```
/workspace/
├── CLAUDE.md                    # 唯一入口
├── .claude/
│   ├── PROTOCOL.md              # 本协议文件
│   ├── workflow/                # 流程配置
│   │   ├── config.yaml
│   │   └── stages/
│   ├── agents/                  # Agent 定义
│   ├── task_states/             # 任务状态
│   ├── task_plans/              # 任务规划
│   ├── subtask_queues/          # 子任务队列
│   ├── task_dependencies.json   # 任务依赖
│   └── execution_logs/          # 执行日志
└── knowledge/                   # 外置知识
```

### 数据流

```
用户输入
    ↓
CLAUDE.md（定义协议）
    ↓
读取 .claude/workflow/config.yaml
    ↓
按 stages 顺序执行
    ↓
每个 stage 读取对应的文件
    ↓
更新状态文件
    ↓
输出结果
```
