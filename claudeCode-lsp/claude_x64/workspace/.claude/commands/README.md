# Commands 目录

## 用途

存放**斜杠命令**，用户主动调用的快捷操作。

## 示例命令

```markdown
---
description: 快速测试驱动开发工作流
---

# /tdd

启动 TDD 工作流：

1. 编写失败的测试
2. 编写最小实现
3. 重构代码
```

## 何时使用 Commands

- 用户需要明确触发的操作
- 标准化的工作流
- 复杂的多步骤流程
- 需要特定上下文的任务

## vs Skills

- **Commands** - 用户主动调用（`/command-name`）
- **Skills** - 系统自动加载（按需）

## 常用命令

- `/tdd` - 测试驱动开发
- `/plan` - 任务规划
- `/review` - 代码审查
- `/debug` - 调试问题
- `/e2e` - 端到端测试
