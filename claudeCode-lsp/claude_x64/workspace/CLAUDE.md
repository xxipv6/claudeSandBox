## ⚠️ 前置强制检查（最高优先级）

**在执行任何操作之前，必须先完成以下步骤！**

### 第一步：任务规划（强制执行）

所有任务都必须先规划，不得跳过！

**规划内容**：
1. 理解用户意图
2. 明确任务边界
3. 识别依赖关系
4. 预估耗时和风险
5. 制定执行计划

**简单任务**：快速规划（1-2 分钟）
**复杂任务**：详细规划（使用 task-planner agent）

### 第二步：用户确认（强制等待）

**规划完成后必须等待用户明确确认！**

- 展示规划结果
- 说明预期耗时
- 说明影响范围
- **等待用户输入"确认"或"开始"**

### 第三步：执行任务（确认后才执行）

**只有在用户明确确认后，才能开始执行任务！**

### 工作流程

所有任务都必须遵循以下流程：

```
用户输入 → 意图识别 → 任务规划 → 用户确认 → 执行任务 → 完成反馈
   ↓           ↓          ↓          ↓          ↓          ↓
 理解意图    映射到命令  调用 planner  等待确认   执行操作   输出结果
```

**关键步骤：意图识别与命令映射**

在开始规划前，先识别用户意图并映射到相应的命令：

| 用户意图 | 关键词 | 使用命令 | 命令调用的 Agents |
|----------|--------|----------|------------------|
| 安全审计 | "审计"、"安全"、"漏洞"、"渗透" | `/security-audit` | security-tester |
| 代码审查 | "审查"、"review"、"检查代码"、"看代码" | `/code-review` | backend-engineer + frontend-engineer（并发） |
| 调试问题 | "调试"、"bug"、"错误"、"不工作" | `/debug` | （使用 debugging skill） |
| 测试功能 | "测试功能"、"测试页面" | `/test` | （使用 testing skill） |
| E2E 测试 | "e2e"、"端到端"、"全部测试"、"前后端测试" | `/e2e` | （并发运行测试） |
| 架构规划 | "规划"、"设计"、"如何实现" | `/plan` | task-planner |

**规则**：
1. **先识别意图** - 检查用户输入是否匹配上述意图
2. **映射到命令** - 如果匹配，使用相应的命令
3. **命令调用 agents** - 命令内部会调用相应的 agents
4. **不匹配时才规划** - 只有在没有匹配命令时，才进行自定义规划

**示例**：
```bash

→ 识别：没有匹配的命令
```
