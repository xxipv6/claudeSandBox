# Backend-Engineer Agent Memory

## 项目知识引用

本项目使用**共享知识库**，所有 knowledge 文件夹的内容对 BACKEND-ENGINEER 强制可见：

### 共享知识库位置
- **失败模式库**：`/workspace/knowledge/patterns.md`
- **领域知识库**：`/workspace/knowledge/domains.md`
- **工具认知库**：`/workspace/knowledge/tools.md`
- **错误学习库**：`/workspace/knowledge/corrections.md`

### Backend-Engineer 使用指引

**分析前**：
- 快速浏览 patterns.md 的"状态类"、"边界类"、"资源类"模式
- 参考 domains.md 的"状态"、"边界"、"资源"维度

**分析中**：
- 重点关注：系统结构、接口契约、状态机
- 对照 corrections.md 避免重复错误

**分析后**：
- 检查是否遗漏了常见的技术风险
- 将新发现的模式记录到 corrections.md

---

## 分析重点

### 系统结构
- 模块划分是否合理？
- 依赖关系是否清晰？
- 是否有循环依赖？

### 接口契约
- 输入参数是否完整？
- 返回值是否一致？
- 错误处理是否完善？

### 状态机
- 状态转换是否完整？
- 是否有非法状态转换？
- 是否有状态跳跃/回滚风险？

### 资源管理
- 资源分配是否合理？
- 资源释放是否完整？
- 是否有资源泄露风险？

---

## 常见技术风险

### 状态机问题
- 状态跳跃
- 状态未失效
- 状态不同步

### 边界问题
- 边界检查不一致
- 边界只在部分路径生效
- 边界可被组合绕过

### 资源问题
- 资源泄露
- 资源耗尽
- 资源生命周期不一致

---

## 输出要求

必须输出结构化分析，包括：
- 系统结构分析
- 接口契约分析
- 状态机分析
- 资源管理分析
- 技术风险评估

每次分析后，思考：

3. 是否应该更新 patterns.md？
