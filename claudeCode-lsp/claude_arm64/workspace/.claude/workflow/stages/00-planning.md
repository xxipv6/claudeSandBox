# Stage 00: Planning（任务规划）

## 目标

启动 task-planner agent，进行任务拆解、优先级排序、依赖识别、模式选择。

## 触发条件

- 用户输入任何任务
- 无论任务大小，都需要规划

## 执行步骤

### 第一步：启动 task-planner

**检查点**：
- [ ] 已读取 `.claude/agents/task-planner.md`
- [ ] 已了解 task-planner 的职责和能力

**启动命令**（使用 Agent tool）：
```
Agent: task-planner
Prompt: [用户输入]
```

### 第二步：等待 task-planner 完成

**task-planner 输出应包含**：
1. 任务理解
2. 任务拆解（1-5 个子任务）
3. 依赖关系（DAG 结构）
4. 执行顺序（拓扑序）
5. 模式选择（标准模式/完整模式）
6. 资源规划（需要的 agents）

**检查点**：
- [ ] task-planner 已完成分析
- [ ] 输出包含所有必需部分
- [ ] 模式选择符合触发条件

### 第三步：生成规划文件（可选）

如果用户确认执行，生成以下文件：

1. **任务规划文件**：`.claude/task_plans/task-{id}.json`
2. **子任务队列文件**：`.claude/subtask_queues/task-{id}.json`
3. **更新任务依赖**：`.claude/task_dependencies.json`
4. **添加到任务队列**：`.claude/task_queue.json`

**检查点**：
- [ ] 文件已创建（如果用户确认）
- [ ] 文件格式符合模板
- [ ] JSON 格式正确

### 第四步：确定下一步

根据 task-planner 的模式选择：

**标准模式**：
- 进入 Stage 01: Task Init

**完整模式**：
- 检查环境变量 `CLAUDE_FULL_MODE=1`
- 如果未设置，拒绝并降级到标准模式
- 如果已设置，进入 Stage 01: Task Init

**检查点**：
- [ ] 已确定执行模式
- [ ] 环境变量已检查（完整模式）
- [ ] 下一步已明确

## 输出格式

### task-planner 输出

task-planner 应按照以下格式输出：

```markdown

### 1. 任务理解
**用户目标**：
**任务边界**：
**成功标准**：

### 2. 任务拆解
**子任务列表**：
...

### 3. 依赖关系
**依赖图（DAG）**：
...

### 4. 执行顺序
**拓扑序**：
...

### 5. 模式选择
**选择的模式**：标准模式 / 完整模式
**选择理由**：

### 6. 资源规划
**需要的 Agents**：
...
```

## 超时处理

- **task-planner 超时**：120 秒
- **超时后操作**：记录超时，询问用户是否继续

## 失败处理

- **task-planner 失败**：记录错误，询问用户是否重试
- **重试次数**：最多 1 次
- **重试后仍失败**：降级到手动规划

- [ ] 模式选择已确定
- [ ] 规划文件已生成（如果用户确认）
- [ ] 下一步已明确
