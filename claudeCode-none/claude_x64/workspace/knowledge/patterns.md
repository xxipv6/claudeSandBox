# Systemic Failure Patterns

这些不是漏洞类型，而是系统失败的方式。

**这是跨 agent 共享的知识库**，所有分析层 agent 都应参考这些模式。

---

## 状态类失败

### 1. 状态跳跃 (State Jump)
**模式**：绕过中间状态，直接到达目标状态

**典型场景**：
- 支付流程中跳过"支付中"直接到"已支付"
- 审批流程中跳过中间审批层级
- 演练状态从"待开始"直接到"已完成"

**检查清单**：
- [ ] 状态转换是否严格遵循状态机？
- [ ] 是否存在可被绕过的中间检查点？
- [ ] 后端是否强制验证状态转换合法性？

### 2. 状态未失效 (State Not Invalidated)
**模式**：应该失效的状态仍然有效

**典型场景**：
- token 过期后仍可使用
- 取消的操作仍能完成
- 删除的资源仍可访问

**检查清单**：
- [ ] 状态是否有明确的失效条件？
- [ ] 失效后是否有强制检查？
- [ ] 是否存在时间窗口漏洞？

### 3. 状态不同步 (State Desynchronization)
**模式**：多个系统间的状态不一致

**典型场景**：
- 数据库与缓存不一致
- 服务间状态同步延迟
- 分布式事务未提交但部分操作已生效

**检查清单**：
- [ ] 分布式状态如何同步？
- [ ] 同步失败如何处理？
- [ ] 是否存在竞态窗口？

## 边界类失败

### 1. 边界检查不一致 (Inconsistent Boundary Check)
**模式**：不同代码路径对同一边界的检查不一致

**典型场景**：
- 前端检查了但后端未检查
- API 层检查了但服务层未检查
- 快速路径未检查但慢速路径检查了

**检查清单**：
- [ ] 边界检查是否在所有路径上一致？
- [ ] 是否存在"快速路径"绕过检查？
- [ ] 前后端边界检查是否对齐？

### 2. 边界只在部分路径生效 (Partial Boundary Enforcement)
**模式**：边界检查只在某些代码路径中生效

**典型场景**：
- 正常流程有检查，异常流程无检查
- 主接口有检查，辅助接口无检查
- 同步操作有检查，异步操作无检查

**检查清单**：
- [ ] 所有代码路径是否都有边界检查？
- [ ] 异常流程是否跳过检查？
- [ ] API 版本间的边界检查是否一致？

### 3. 边界可被组合绕过 (Composable Boundary Bypass)
**模式**：多个合法操作组合后绕过边界

**典型场景**：
- 分别合法的两个操作组合后非法
- 分批操作绕过单次限制
- 时序组合绕过状态检查

**检查清单**：
- [ ] 操作组合后的边界是否仍有效？
- [ ] 是否存在组合攻击路径？
- [ ] 批量操作是否有额外的边界检查？

## 信任类失败

### 1. 信任假设错误 (Incorrect Trust Assumption)
**模式**：系统信任了不该信任的东西

**典型场景**：
- 信任前端传递的用户 ID
- 信任不可变的头部数据
- 信任内部调用（未验证权限）

**检查清单**：
- [ ] 系统信任了哪些输入？
- [ ] 这些输入是否可被伪造？
- [ ] 是否有最小权限原则？

### 2. 信任未随状态变化 (Trust Not Updated)
**模式**：状态变化后信任关系未更新

**典型场景**：
- 用户降权后仍能访问高权限资源
- 资源共享后取消共享但仍可访问
- 角色变更后权限未立即生效

**检查清单**：
- [ ] 状态变化时信任关系是否更新？
- [ ] 是否存在信任缓存？
- [ ] 权限变更的传播延迟是多少？

### 3. 跨边界信任传递 (Cross-Boundary Trust Propagation)
**模式**：信任关系跨越安全边界传递

**典型场景**：
- 内部系统的信任传递到外部系统
- 高权限上下文的信任传递到低权限上下文
- 跨服务的信任传递未重新验证

**检查清单**：
- [ ] 跨边界调用是否重新验证信任？
- [ ] 是否存在信任传递链？
- [ ] 信任传递的边界在哪里？

## 时间类失败

### 1. 顺序依赖 (Order Dependency)
**模式**：系统依赖操作顺序但未强制

**典型场景**：
- 必须先 A 后 B，但可并发执行
- 初始化依赖未满足但继续执行
- 事件处理顺序可被操纵

**检查清单**：
- [ ] 操作顺序是否强制？
- [ ] 是否存在顺序检查？
- [ ] 并发操作如何处理顺序？

### 2. 竞态条件 (Race Condition)
**模式**：多线程/多进程间的时序问题

**典型场景**：
- Check-then-act 竞态
- Double-fetch 竞态
- Time-of-check vs Time-of-use (TOCTOU)

**检查清单**：
- [ ] 是否存在共享状态？
- [ ] 共享状态的访问是否原子？
- [ ] 是否存在检查与使用之间的时间窗口？

### 3. 时间窗口 (Time Window)
**模式**：某些操作必须在特定时间内完成但未强制

**典型场景**：
- 验证码无限期有效
- 临时凭证未过期
- 会话超时时间过长

**检查清单**：
- [ ] 时间敏感操作是否有超时？
- [ ] 超时后是否自动失效？
- [ ] 时间窗口是否可被延长？

## 资源类失败

### 1. 生命周期不一致 (Lifecycle Mismatch)
**模式**：资源生命周期与状态生命周期不一致

**典型场景**：
- 资源已释放但状态仍引用
- 状态已失效但资源未释放
- 资源与状态的清理顺序错误

**检查清单**：
- [ ] 资源生命周期是否明确？
- [ ] 资源与状态的清理是否同步？
- [ ] 是否存在资源泄露？

### 2. 释放与状态脱钩 (Release Decoupled from State)
**模式**：资源释放不依赖于状态变化

**典型场景**：
- 资源释放但状态未更新
- 状态更新但资源未释放
- 异常情况下资源未释放

**检查清单**：
- [ ] 资源释放是否与状态绑定？
- [ ] 异常情况下资源如何释放？
- [ ] 是否使用 finally/context manager？

---

## 组合类失败

### 1. 合法操作的非法组合 (Illegal Combination of Legal Operations)
**模式**：单独合法的操作组合后非法

**典型场景**：
- 分批转账绕过单次限制
- 多个低权限操作组合获得高权限
- 时序操作绕过状态检查

**检查清单**：
- [ ] 操作组合是否检查？
- [ ] 是否存在原子性保证？
- [ ] 批量操作是否有额外检查？

### 2. 跨边界组合 (Cross-Boundary Composition)
**模式**：跨系统/边界的操作组合产生问题

**典型场景**：
- 跨服务的操作组合绕过限制
- 跨 API 版本的组合攻击
- 跨租户的资源组合

**检查清单**：
- [ ] 跨边界操作是否有全局检查？
- [ ] 是否存在跨边界的原子性保证？
- [ ] 边界间的信任关系如何？

---

## 安全研究特定模式

### 1. 信息泄露模式
- 错误消息泄露系统细节
- 调试信息在生产环境暴露
- 枚举攻击（用户 ID、订单 ID 等）
- 时间侧信道（响应时间差异）

### 2. 权限提升模式
- 垂直越权：普通用户获得管理员权限
- 水平越权：用户 A 访问用户 B 的资源
- 跨租户越权：租户 A 访问租户 B 的资源

### 3. 业务逻辑漏洞模式
- 价格操纵
- 优惠券滥用
- 积分/限额绕过
- 工作流 bypass

### 4. 状态机绕过模式
- 状态回滚
- 状态跳跃
- 并发状态转换
- 状态不一致利用

## 各 Agent 使用指引

### product-manager
关注：状态类、组合类
- 检查业务流程中的状态跳跃
- 检查业务操作的组合风险

### backend-engineer
关注：状态类、边界类、资源类
- 检查状态机实现
- 检查边界检查一致性
- 检查资源生命周期

### frontend-engineer
关注：边界类、信任类
- 检查前端输入面边界
- 检查前端与后端的信任关系

### qa-engineer
关注：边界类、时间类
- 枚举所有边界场景
- 检查竞态条件
- 检查时间窗口

### security-tester
关注：信任类、组合类、安全研究特定模式
- 构建攻击路径
- 检查权限提升
- 检查业务逻辑漏洞

---

## 使用方法

1. **分析前**：快速浏览这个文件，激活相关模式
2. **分析中**：对照检查清单，逐项验证
3. **分析后**：将发现的模式记录到输出中

**记住**：这些模式是"警示标志"，不是"漏洞清单"。看到模式时要深入分析具体场景。
