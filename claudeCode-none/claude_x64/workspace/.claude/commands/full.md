# 完整开发流程

## 何时使用

- 新功能开发
- 复杂任务实现
- 需要系统化规划的场景

## 开发流程

### 阶段 1：规划

1. **启动 task-planner**
   ```bash
   读取：.claude/agents/task-planner.md
   执行：task-planner agent
   ```

2. **生成执行计划**
   - 任务分解
   - 依赖分析
   - 风险识别
   - 时间估算

3. **确认计划**
   - 与用户确认计划
   - 调整优先级
   - 明确交付物

### 阶段 2：分析（并发）

1. **启动分析 agents**
   ```bash
   并发执行：
   - product-manager（需求分析）
   - backend-engineer（后端架构）
   - frontend-engineer（前端实现）
   - security-tester（安全分析）
   ```

2. **收集分析结果**
   - 需求文档
   - 架构设计
   - API 设计
   - 安全考虑

3. **综合设计**
   - 整合各视角
   - 解决冲突
   - 确定技术方案

### 阶段 3：实现

1. **启动 dev-coder**
   ```bash
   读取：.claude/agents/dev-coder.md
   执行：dev-coder agent
   ```

2. **实现功能**
   - 编写代码
   - 遵循规范
   - 添加注释

3. **编写测试**
   - 单元测试
   - 集成测试
   - 边界测试

### 阶段 4：验证

1. **代码审查**
   ```bash
   执行：/code-review
   ```

2. **安全审查**
   ```bash
   执行：/secure-review
   ```

3. **测试验证**
   ```bash
   执行：/test
   ```

### 阶段 5：交付

1. **文档更新**
   - API 文档
   - README
   - 变更日志

2. **提交代码**
   ```bash
   git add .
   git commit -m "feat: ..."
   ```

3. **创建 PR**（可选）
   ```bash
   gh pr create
   ```

## 使用示例

```bash
# 完整功能开发
/full "实现用户认证系统，包括登录、注册、密码重置"

# API 开发
/full "添加 REST API 用于用户管理"

# 前端功能
/full "实现用户个人资料页面"
```

## 时间估算

| 阶段 | 时间 |
|------|------|
| 规划 | 10-15% |
| 分析 | 15-20% |
| 实现 | 40-50% |
| 验证 | 15-20% |
| 交付 | 5-10% |

## 注意事项

- 每个阶段都要确认
- 发现问题及时调整
- 保持沟通
- 记录决策
