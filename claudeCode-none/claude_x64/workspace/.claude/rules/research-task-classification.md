# Research Task Classification

## 快速分类

### 简单任务
满足任一条件可直接执行：
- 只读查询
- 单步验证
- 单一函数 / 单一接口检查
- 单命令完成的检查任务
- 预计几分钟内完成

### 中等复杂度任务
满足任一条件，直接进入研究或验证；如出现分叉再升级：
- 单点漏洞分析
- 已知漏洞复现
- 范围明确的代码审计
- 单一攻击面测试
- PoC 路径明确

### 高复杂度任务
满足任一条件，先 Plan，再决定 Agent Strategy：
- 系统级审计
- 深度逆向
- 攻击链构建
- 多攻击面分析
- 研究路径明显不确定
- 预计涉及大量文件或多阶段验证

---

## 升级条件

执行中出现以下情况，立即重新评估 Agent Strategy；如果已经不是单一路径验证，而是两条以上可独立推进的证据分支，切换为 Multi-Agent。如果很快重新收敛回单一路径，则保持或降回 Single-Agent：
- 原本单点问题演变成多路径研究
- 需要逆向未知协议 / 格式
- 需要静态 + 动态联合验证
- 需要多个 Agent 并行收集证据
- 原本明确的验证路径失效
- reverse + audit、reverse + dynamic validation、reverse + skeptic 反证、reverse + JS / protocol 独立分支同时出现

---

## 降级条件

探索后如果发现：
- 只是简单配置问题
- 只是局部代码问题
- 不再需要多阶段验证
- 不再需要多 Agent

则降回中等复杂度或简单任务，直接执行。

---

## 默认处理

- 简单任务：直接执行
- 中等复杂度：直接研究 / 验证；出现分叉再升级
- 高复杂度：先 Plan，再决定 Single-Agent 或 Multi-Agent

关键点：**快速判断，动态升级，不要把所有任务都按高复杂度处理。**
