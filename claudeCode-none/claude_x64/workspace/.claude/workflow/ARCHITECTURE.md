# 系统架构说明

## 一、核心模式（单一系统，自动分级）

```
用户输入
    ↓
Claude 理解意图
    ↓
强制输出 MODE DECISION
    ↓
自动判断复杂度
    ↓
┌─────────┬─────────┬─────────┐
│  快速    │  标准    │  完整    │
│  模式    │  模式    │  模式    │
└─────────┴─────────┴─────────┘
```

---

## 二、强制入口规则（最高优先级）

### MODE DECISION
在执行任何操作前，必须先输出：

```
MODE DECISION:
- Detected signals:
- Selected mode: [快速 | 标准 | 完整]
- Reason:
```

**效果**：防止 Claude "先做再说"，确保模式明确。

---

## 三、三种执行模式对比

### 快速模式（Quick Mode）
```
输入：单文件 / 明确修复 / 用户说"快速"
  ↓
直接执行（0 个 agents）
  ↓
输出结果
```

| 特性 | 值 |
|-----|-----|
| Agents | 0 个 |
| 状态文件 | 无 |
| 计划确认 | 否 |
| 适用 | 单文件修改、明确 bug 修复 |

---

### 标准模式（Standard Mode）← 默认、安全缓冲区
```
输入：多文件 / 需要分析 / 用户说"分析" / 不确定
  ↓
读取 standard-mode.md
  ↓
制定简要计划（2-3 步）
  ↓
询问用户确认
  ↓
启动 1-2 个 agents（顺序）
  ↓
输出简洁报告
```

| 特性 | 值 |
|-----|-----|
| Agents | 1-2 个（顺序） |
| 状态文件 | 可选 |
| 计划确认 | 是（简要） |
| 适用 | 项目分析、功能实现、任何不确定 |

---

### 完整模式（Full Mode）
```
输入：架构级 + "完整分析" + CLAUDE_FULL_MODE=1
  ↓
读取 full-mode.md
  ↓
制定详细计划（6 阶段）
  ↓
询问用户确认
  ↓
创建分支 + 状态文件
  ↓
启动 2-4 个 agents（顺序）
  ↓
执行 Quality Gate
  ↓
合并分支
```

| 特性 | 值 |
|-----|-----|
| Agents | 2-4 个（顺序） |
| 状态文件 | 完整 |
| 计划确认 | 是（详细） |
| Git | 创建分支 |
| 触发 | 环境变量 + 用户明确要求 |

**拒绝条件**：环境变量未设置 → 明确拒绝，降级到标准模式

## 四、判断逻辑（保守策略）

```python
def detect_mode(user_input, context):
    # 快速模式（明确信号）
    if is_single_file(context) or has_explicit_fix(user_input) or user_says_quick(user_input):
        return "quick"

    # 完整模式（明确授权）
    if is_architecture_level(context) and user_says_full_analysis(user_input) and env_has_full_mode():
        return "full"

    # 标准模式（默认，安全缓冲区）
    return "standard"  # 任何不确定都到这里
```

**核心原则**：不确定 → 标准模式

## 七、核心设计理念

| 维度 | 之前（660 行） | 现在（96 行） |
|-----|---------------|--------------|
| **驱动方式** | 配置驱动 | 意图驱动 |
| **复杂度** | 660 行规则 | 96 行原则 |
| **Agent 数量** | 固定 4-9 个并发 | 0-4 个顺序 |
| **模式选择** | 两阶段强制 | 自动判断 |
| **状态文件** | 强制创建 | 按需创建 |
| **用户确认** | 形式化 | 实用化 |
| **优先级** | 证明合规 | 解决问题 |
| **安全缓冲** | 无 | 标准模式 |

---

## 八、关键改进

### 1. 强制入口
- MODE DECISION 防止"先做再说"
- 硬门槛，不可绕过

### 2. 默认拒绝
- 完整模式需要环境变量
- 未设置时明确拒绝

### 3. 保守判断
- 任何不确定 → 标准模式
- 标准模式是安全缓冲区

### 4. 顺序执行
- Agents 顺序执行，不并发
- Claude 的并发是叙事级的

### 5. 禁止自动 skill
- 不自动使用调试技能
- 除非用户明确要求

---

## 九、为什么这样设计？

### 问题 1：660 行无法执行
**解决**：精简到 96 行核心原则

### 问题 2：配置驱动不工作
**解决**：改为意图驱动（Claude 擅长理解意图）

### 问题 3：并发 agents 不可控
**解决**：改为顺序执行

### 问题 4：过度形式化
**解决**：优先解决问题，不是证明合规

### 问题 5：没有安全缓冲
**解决**：标准模式作为默认和安全缓冲区
