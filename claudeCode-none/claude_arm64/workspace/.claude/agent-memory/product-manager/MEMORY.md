# Product-Manager Agent Memory

## 项目知识引用

本项目使用**共享知识库**，所有 knowledge 文件夹的内容对 PRODUCT-MANAGER 强制可见：

### 共享知识库位置
- **失败模式库**：`/workspace/knowledge/patterns.md`
- **领域知识库**：`/workspace/knowledge/domains.md`
- **工具认知库**：`/workspace/knowledge/tools.md`
- **错误学习库**：`/workspace/knowledge/corrections.md`

### Product-Manager 使用指引

**分析前**：
- 快速浏览 patterns.md 的"状态类"、"组合类"模式
- 参考 domains.md 的"状态"、"组合"维度

**分析中**：
- 重点关注：业务目标、需求完整性、隐含假设
- 对照 corrections.md 避免重复错误

**分析后**：
- 检查是否遗漏了常见的业务逻辑漏洞
- 将新发现的模式记录到 corrections.md

---

## 分析重点

### 需求完整性
- 是否有未明确说明的需求？
- 是否有隐含的假设？
- 是否有边界条件未覆盖？

### 业务目标
- 这个功能要解决什么问题？
- 成功标准是什么？
- 是否有冲突的目标？

### 隐含假设
- 假设了什么用户行为？
- 假设了什么技术能力？
- 假设了什么环境条件？

---

## 常见业务逻辑漏洞

### 价格操纵
- 负数价格
- 极小价格
- 价格竞态

### 优惠券滥用
- 叠加使用
- 转赠使用
- 过期后使用

### 积分/限额绕过
- 分批操作
- 并发操作
- 时间窗口

---

## 输出要求

必须输出结构化分析，包括：
- 业务目标分析
- 需求完整性检查
- 隐含假设识别
- 业务逻辑漏洞识别
- 风险评估

---

## 持续改进

每次分析后，思考：
1. 是否发现了新的业务逻辑漏洞模式？
2. 是否应该更新 corrections.md？
3. 是否应该更新 patterns.md？
