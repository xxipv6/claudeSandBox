# Web 白盒安全审计

## 何时启用

- 用户请求 Web 应用安全审计
- 检测到 Web 应用代码（Spring / Django / Express / Gin 等）
- 需要识别注入漏洞、越权、逻辑缺陷
- 需要分析跨接口、跨状态联动风险
- 代码审查涉及安全问题

---

## 审计核心原则

- 不只找"漏洞点"，而是**还原真实执行模型**
- 不只看"有没有校验"，而是看**校验是否完整、是否可组合**
- 不只看"单接口"，而是看**跨接口、跨状态联动**
- **越权作为独立主线贯穿全流程**

---

## 完整审计流程（8 阶段）

### 1️⃣ 系统执行模型与信任边界建模

**目标**：明确请求如何进入系统、在哪些地方被信任或放行。

**审计要点**：
- **外部入口**：
  - HTTP API
  - RPC
  - MQ / 定时任务
- **信任边界**：
  - 用户 → Web
  - Web → 内部服务
  - 服务 → 第三方

**输出**：
- 请求入口 → 业务代码的执行路径图
- 外部可控 vs 内部可信边界标注

---

### 2️⃣ 依赖与框架层白盒分析（SCA + 能力审计）

**目标**：识别"危险能力"是否被引入并暴露给用户路径。

**重点关注**：
- 反序列化（JSON / XML / 自定义）
- 表达式解析（SpEL / OGNL / 模板）
- 动态执行（脚本 / 反射 / 插件）
- 网络能力（HTTP Client / SSRF）

**核心问题**：
- 这些能力是否在**用户可控路径**上？

---

### 3️⃣ 请求执行链还原

**目标**：还原一次请求从入口到业务副作用的完整执行链。

**审计要点**：
- Filter / Middleware 顺序
- Interceptor / Hook 是否全覆盖
- AOP / 注解是否存在遗漏
- 安全逻辑执行顺序是否正确

**重点风险**：
- 某些路由绕过安全链
- 特殊 Method / Header 绕过
- 注解只作用于接口、不作用于实现

**输出**：
- 标准执行链
- 绕过 / 异常执行路径清单

---

### 4️⃣ 路由全量枚举与能力分类

**目标**：明确系统对外暴露的全部能力。

**对每个路由标注**：
- 是否对外可访问
- 是否需要认证
- 是否有副作用（写操作）
- 权限敏感度（低 / 中 / 高）

**特别关注**：
- 管理接口
- Debug / Test 路由
- 同一路由不同 HTTP Method

---

### 5️⃣ 控制器与业务流程级白盒审计

**目标**：识别输入 → 校验 → 状态 → 副作用的完整链路。

**对每个路由强制回答**：
- 输入来自哪里？
- 校验在哪里？是否完整？
- 状态在哪里改变？
- 副作用是什么？

**重点风险**：
- 校验不一致
- 校验依赖外部状态
- Service 层缺失校验

---

### 6️⃣ 越权专项审计

#### 6.1 权限模型梳理
明确：
- 角色模型（RBAC / ABAC / 混合）
- 权限粒度（接口 / 资源 / 状态）

#### 6.2 权限控制点枚举
系统性查找：
- Filter / Interceptor
- AOP / 注解
- Controller 内判断
- Service 层校验
- DAO 层隐式权限（userId 过滤）

#### 6.3 路由级越权
检查：
- 是否缺认证
- 是否缺角色校验
- 是否默认放行

#### 6.4 资源级越权（高发）
**重点参数**：`id / uid / resourceId`
- 必须校验资源是否存在
- 必须校验是否属于当前用户
- 必须校验当前用户是否有操作权限

#### 6.5 状态 + 权限联动越权
检查：
- 权限是否依赖状态
- 状态是否可被低权限接口修改
- 状态与权限是否解耦

#### 6.6 跨接口越权组合
**典型模式**：
- A 创建资源（低权限）
- B 修改状态（校验不严）
- C 执行高权限操作（只看状态）

---

### 7️⃣ 状态机与跨路由联动分析

**目标**：识别业务状态被滥用的可能性。

**审计要点**：
- 列出所有关键状态字段
- 标注：
  - 创建接口
  - 修改接口
  - 依赖接口
- 打乱设计者预期调用顺序

---

### 8️⃣ 攻击路径建模与验证

**假设**：我是最低权限用户

**构造**：
- 合法请求序列
- 合法参数
- 非预期结果

**确认**：
- 是否真实可利用
- 是否影响业务安全

---

## 漏洞输出标准

每个问题至少包含：

- **漏洞类型**：水平越权 / 垂直越权 / 逻辑缺陷 / 注入漏洞
- **触发路径**：接口 A → B → C
- **执行链说明**：完整的请求调用链
- **状态变化说明**：状态如何被修改
- **风险等级**：高 / 中 / 低
- **修复建议**：原则级指导

---

## 常见白盒"假安全"模式（重点识别）

- ❌ 只在 Controller 校验 → Service 可被复用绕过
- ❌ 只校验角色，不校验资源 → 水平越权
- ❌ 异常路径无权限校验 → 绕过风险
- ❌ 批量接口只校验第一个元素 → 部分越权
- ❌ 校验依赖外部状态 → 状态竞争
- ❌ 状态与权限耦合 → 联动越权

---

## 适用对象

- Web 应用（单体 / 微服务）
- REST API / BFF / 管理后台
- 框架不限：Spring / Django / Express / Gin 等
