# /debug - 调试命令

## 执行流程

1. **理解问题**
   - 收集错误信息
   - 分析堆栈跟踪
   - 确定问题范围

2. **定位根因**
   - 使用 debugging skill
   - 添加日志/断点
   - 验证假设

3. **制定修复方案**
   - 调用 planner agent
   - 生成修复计划

4. **实施修复**
   - 调用 tdd-guide（编写失败测试）
   - 调用 dev（实施修复）
   - 验证修复

5. **代码审查**
   - 调用 reviewer
   - 确认修复正确

## 使用场景

- ✅ 功能不按预期工作
- ✅ 出现错误或异常
- ✅ 性能问题
- ✅ 需要理解现有代码行为
