# 标准模式示例

## 触发条件
- 多文件操作（2-10 个文件）
- 需要分析/理解代码
- 用户说"分析"、"实现"
- 任何不确定的情况

## 示例 1：项目分析
```
用户：分析这个项目的架构

Claude：
1. 制定简要计划：
   - 读取主要文件
   - 启动 product-manager 分析架构
   - 输出分析报告

2. 询问用户：是否按这个计划执行？

3. 用户确认后：
   - 读取文件
   - 启动 product-manager agent
   - 合并结果
   - 输出简洁的分析报告
```

## 示例 2：功能实现
```
用户：实现一个用户登录功能

Claude：
1. 制定简要计划：
   - 分析现有代码结构
   - 启动 backend-engineer 分析后端
   - 启动 dev-coder 实现代码
   - 测试验证

2. 询问用户：是否按这个计划执行？

3. 用户确认后：
   - 按步骤执行
   - 每步完成后汇报进度
```
