## 一、快速模式测试（单文件修改）

### 测试命令
```
修复 CLAUDE.md 第 5 行的错别字
```

### 期望行为
- ✅ 直接读取文件并修改
- ✅ 不启动任何 agent
- ✅ 不创建状态文件
- ✅ 不询问计划确认
- ✅ 直接输出修复结果

### 如果失败
- Claude 启动了 agent → 系统判断有误
- Claude 创建了状态文件 → 系统判断有误
- Claude 询问计划确认 → 系统判断有误

## 二、标准模式测试（简单分析）

### 测试命令
```
分析 .claude/agents/ 目录下的 agent 定义
```

### 期望行为
- ✅ 先制定简要计划
- ✅ 询问用户确认
- ✅ 用户确认后执行
- ✅ 启动 1-2 个 agents（顺序执行）
- ✅ 输出简洁的分析报告（≤ 500 行）

### 如果失败
- Claude 直接执行不询问 → 违反标准模式规则
- Claude 启动 4+ agents → 违反顺序执行原则
- Claude 输出 500+ 行报告 → 过度形式化

## 五、成功标准

| 标准 | 要求 |
|-----|------|
| 快速模式 | 直接执行，无 agent，无状态文件 |
| 标准模式 | 简要计划 + 确认 + 1-2 agents（顺序） |
| 完整模式 | 详细计划 + 确认 + 环境变量检查 |
| 配置文件 | workflow/ 不作为执行配置 |
| 行为优先级 | 解决问题 > 证明合规 |
