## ⚠️ 前置强制检查（最高优先级）

**在执行任何操作之前，必须先完成以下步骤！**

### 1. 任务规划（强制执行）
- 理解用户意图
- 明确任务边界
- 识别依赖关系
- 预估耗时和风险
- 制定执行计划

### 2. 用户确认（强制等待）
- 展示规划结果
- 说明预期耗时和影响范围
- **等待用户输入"确认"或"开始"**

### 3. 执行任务（确认后才执行）
**只有在用户明确确认后，才能开始执行任务！**

## 核心命令

```bash
/security-audit     # 安全审计（Web 白盒 + IoT）
/code-review        # 代码审查（前后端并发）
/debug              # 调试问题
/test               # 功能测试
/e2e                # E2E 测试（前端 + 后端，并发）
```

---

## 意图识别与命令映射

**优先级：意图识别 → 命令映射 → 执行**

| 用户意图 | 关键词 | 使用命令 |
|----------|--------|----------|
| 安全审计 | 审计、安全、漏洞、渗透 | `/security-audit` |
| 代码审查 | 审查、review、检查代码 | `/code-review` |
| 调试问题 | 调试、bug、错误 | `/debug` |
| 测试功能 | 测试功能、测试页面 | `/test` |
| E2E 测试 | e2e、端到端、全部测试 | `/e2e` |
| 架构规划 | 规划、设计、如何实现 | `/plan` |

**执行规则**：
1. 先识别用户意图是否匹配上表
2. 如果匹配，使用相应的命令
3. 如果不匹配，进行自定义任务规划

---

## 核心原则

1. **规划优先** - 所有任务都必须先规划，再执行
2. **理解优先** - 先理解意图，再执行
3. **确认重要** - 复杂任务先给计划，等确认
4. **高度自治** - 缺什么就装什么，无需确认
5. **按需调用** - 根据任务选择合适的命令/skill/agent

**高度自治**：
- ✅ 缺少依赖 → 自动安装
- ✅ 缺少配置 → 自动创建
- ✅ 缺少工具 → 自动安装
- ✅ 服务未运行 → 自动启动
- ❌ 不需要问"是否安装/创建/启动"

## 全局禁止

1. ❌ **跳过规划直接执行**
2. ❌ 跳过用户确认就执行大规模修改
3. ❌ 违反安全规则（硬编码密钥、缺少输入验证等）
4. ❌ 过度形式化（输出冗长的分析报告）
5. ❌ **过度询问** - 环境准备不需要反复确认
