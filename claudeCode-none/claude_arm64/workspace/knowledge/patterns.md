# Systemic Failure Patterns

这些不是漏洞类型，而是系统失败的方式。

**这是跨 agent 共享的知识库**，所有分析层 agent 都应参考这些模式。

---

## 状态类失败

### 1. 状态跳跃 (State Jump)
**模式**：绕过中间状态，直接到达目标状态

**典型场景**：
- 支付流程中跳过"支付中"直接到"已支付"
- 审批流程中跳过中间审批层级
- 流程状态从"待开始"直接到"已完成"

**检查清单**：
- [ ] 状态转换是否严格遵循状态机？
- [ ] 是否存在可被绕过的中间检查点？
- [ ] 后端是否强制验证状态转换合法性？

### 2. 状态未失效 (State Not Invalidated)
**模式**：应该失效的状态仍然有效

**典型场景**：
- token 过期后仍可使用
- 取消的操作仍能完成
- 删除的资源仍可访问

**检查清单**：
- [ ] 状态是否有明确的失效条件？
- [ ] 失效后是否有强制检查？
- [ ] 是否存在时间窗口漏洞？

### 3. 状态不同步 (State Desynchronization)
**模式**：多个系统间的状态不一致

**典型场景**：
- 数据库与缓存不一致
- 服务间状态同步延迟
- 分布式事务未提交但部分操作已生效

**检查清单**：
- [ ] 分布式状态如何同步？
- [ ] 同步失败如何处理？
- [ ] 是否存在竞态窗口？

### 4. 状态死锁 (State Deadlock)
**模式**：系统陷入无法继续执行的状态

**典型场景**：
- 两个或多个组件互相等待对方释放资源
- 状态转换循环依赖
- 异常处理路径缺少状态恢复

**检查清单**：
- [ ] 是否存在循环依赖的状态转换？
- [ ] 异常情况下是否有状态恢复机制？
- [ ] 是否有超时机制打破死锁？

## 边界类失败

### 1. 边界检查不一致 (Inconsistent Boundary Check)
**模式**：不同代码路径对同一边界的检查不一致

**典型场景**：
- 前端检查了但后端未检查
- API 层检查了但服务层未检查
- 快速路径未检查但慢速路径检查了

**检查清单**：
- [ ] 边界检查是否在所有路径上一致？
- [ ] 是否存在"快速路径"绕过检查？
- [ ] 前后端边界检查是否对齐？

### 2. 边界只在部分路径生效 (Partial Boundary Enforcement)
**模式**：边界检查只在某些代码路径中生效

**典型场景**：
- 正常流程有检查，异常流程无检查
- 主接口有检查，辅助接口无检查
- 同步操作有检查，异步操作无检查

**检查清单**：
- [ ] 所有代码路径是否都有边界检查？
- [ ] 异常流程是否跳过检查？
- [ ] API 版本间的边界检查是否一致？

### 3. 边界可被组合绕过 (Composable Boundary Bypass)
**模式**：多个合法操作组合后绕过边界

**典型场景**：
- 分别合法的两个操作组合后非法
- 分批操作绕过单次限制
- 时序组合绕过状态检查

**检查清单**：
- [ ] 操作组合后的边界是否仍有效？
- [ ] 是否存在组合攻击路径？
- [ ] 批量操作是否有额外的边界检查？

## 信任类失败

### 1. 信任假设错误 (Incorrect Trust Assumption)
**模式**：系统信任了不该信任的东西

**典型场景**：
- 信任客户端发送的数据
- 信任内部服务之间无需验证
- 信任环境变量或配置文件

**检查清单**：
- [ ] 系统信任了什么？
- [ ] 信任是否可被伪造？
- [ ] 是否遵循零信任原则？

### 2. 信任传递链断裂 (Broken Trust Chain)
**模式**：信任在传递过程中丢失或被篡改

**典型场景**：
- token 传递过程中未验证签名
- 权限在服务间传递时未验证
- 上下文信息在跨服务调用时丢失

**检查清单**：
- [ ] 信任如何在系统间传递？
- [ ] 传递过程中是否验证？
- [ ] 是否存在传递链断裂的风险？

### 3. 信任边界模糊 (Ambiguous Trust Boundary)
**模式**：系统的信任边界不清晰

**典型场景**：
- 内部/外部边界不明确
- 租户/用户边界混淆
- 管理/运行时权限边界重叠

**检查清单**：
- [ ] 信任边界在哪里？
- [ ] 边界是否清晰定义？
- [ ] 是否存在边界模糊导致的风险？

## 时间类失败

### 1. 竞态条件 (Race Condition)
**模式**：系统行为取决于操作的时序

**典型场景**：
- TOCTOU（检查-使用时间窗口）
- 并发操作导致状态不一致
- 异步操作完成顺序不确定

**检查清单**：
- [ ] 是否存在共享资源的并发访问？
- [ ] 检查和使用之间是否有时间窗口？
- [ ] 是否使用锁或其他同步机制？

### 2. 时间窗口漏洞 (Time Window Vulnerability)
**模式**：系统在特定时间窗口内存在漏洞

**典型场景**：
- 状态转换过程中的时间窗口
- 缓存失效与更新之间的时间窗口
- 异步操作完成前的时间窗口

**检查清单**：
- [ ] 系统在哪些时间点最脆弱？
- [ ] 时间窗口是否可被恶意利用？
- [ ] 是否有机制缩短或保护时间窗口？

### 3. 时序假设错误 (Incorrect Timing Assumption)
**模式**：系统假设事件会按特定时序发生

**典型场景**：
- 假设消息按顺序到达
- 假设异步操作快速完成
- 假设回调函数一定会被调用

**检查清单**：
- [ ] 系统是否假设了特定的时序？
- [ ] 如果时序错乱会怎样？
- [ ] 是否有机制处理乱序？

## 资源类失败

### 1. 资源泄漏 (Resource Leak)
**模式**：资源未被正确释放

**典型场景**：
- 内存泄漏
- 连接泄漏（数据库、网络）
- 文件句柄泄漏

**检查清单**：
- [ ] 所有资源获取都有对应的释放吗？
- [ ] 异常情况下资源是否正确释放？
- [ ] 是否有资源监控机制？

### 2. 资源耗尽 (Resource Exhaustion)
**模式**：资源被耗尽导致拒绝服务

**典型场景**：
- 内存耗尽
- 连接池耗尽
- 磁盘空间耗尽

**检查清单**：
- [ ] 资源是否有使用限制？
- [ ] 资源耗尽时如何处理？
- [ ] 是否有资源监控和告警？

### 3. 资源竞争 (Resource Contention)
**模式**：多个组件竞争同一资源

**典型场景**：
- 数据库连接竞争
- 缓存竞争
- 锁竞争

**检查清单**：
- [ ] 哪些资源会被多个组件访问？
- [ ] 如何协调资源访问？
- [ ] 是否存在性能瓶颈？

---

## 组合类失败

### 1. 分布式系统失败 (Distributed System Failure)
**模式**：分布式系统特有的失败模式

**典型场景**：
- 网络分区导致数据不一致
- 部分节点失败导致系统不可用
- 分布式事务超时

**检查清单**：
- [ ] 系统如何处理网络分区？
- [ ] 部分失败如何恢复？
- [ ] 是否有降级机制？

### 2. 级联失败 (Cascading Failure)
**模式**：一个组件的失败导致其他组件失败

**典型场景**：
- 下游服务失败导致上游服务积压
- 缓存失效导致数据库过载
- 依赖服务失败导致链式反应

**检查清单**：
- [ ] 组件间的依赖关系是什么？
- [ ] 一个组件失败会影响哪些其他组件？
- [ ] 是否有隔离和降级机制？

### 3. Byzantine 失败 (Byzantine Failure)
**模式**：组件行为不一致或恶意

**典型场景**：
- 节点发送冲突的信息
- 节点部分响应
- 节点响应延迟

**检查清单**：
- [ ] 系统如何处理不一致的信息？
- [ ] 是否有多数表决机制？
- [ ] 是否有拜占庭容错机制？

## 分析层特定失败

### 1. 意图识别失败 (Intent Recognition Failure)
**模式**：系统错误判断用户意图

**典型场景**：
- 将复杂任务误判为简单任务
- 将分析需求误判为执行需求
- 未识别隐含的分析需求

**检查清单**：
- [ ] 任务复杂度判断是否准确？
- [ ] 是否考虑了隐含需求？
- [ ] 用户意图是否被正确理解？

### 2. 分级调度失败 (Graded Scheduling Failure)
**模式**：错误选择 agent 数量或类型

**典型场景**：
- 简单任务调度过多 agent
- 复杂任务调度过少 agent
- 未调度必要的专家 agent

**检查清单**：
- [ ] 任务复杂度判断是否准确？
- [ ] 是否选择了合适的专家？
- [ ] agent 数量是否合适？

### 3. 证据验证失败 (Evidence Verification Failure)
**模式**：接受无效证据或拒绝有效证据

**典型场景**：
- 将主观判断当作事实
- 未验证证据来源
- 忽视有效证据

**检查清单**：
- [ ] 证据是否符合有效性标准？
- [ ] 证据来源是否可靠？
- [ ] 是否有足够的证据支持结论？

## 执行层特定失败

### 1. 代码生成失败 (Code Generation Failure)
**模式**：生成的代码存在缺陷

**典型场景**：
- 代码逻辑错误
- 缺少错误处理
- 边界条件未处理

**检查清单**：
- [ ] 代码是否通过语法检查？
- [ ] 是否有完整的错误处理？
- [ ] 边界条件是否处理？

### 2. 上下文理解失败 (Context Understanding Failure)
**模式**：轻量分析未能正确理解上下文

**典型场景**：
- 读取文件不完整
- 理解需求有偏差
- 问题定位不准确

**检查清单**：
- [ ] 是否读取了相关文件？
- [ ] 需求理解是否正确？
- [ ] 问题定位是否准确？

### 3. 优化建议失败 (Optimization Suggestion Failure)
**模式**：优化建议不准确或不实用

**典型场景**：
- 优化方向错误
- 优化建议不可行
- 优化收益被夸大

**检查清单**：
- [ ] 优化方向是否正确？
- [ ] 优化建议是否可行？
- [ ] 优化收益是否真实？
