# Unified Security Problem Space

本文件定义安全工程的"问题本质"，不区分 Web / Binary / IoT / Protocol。

所有系统都服从同一套分析维度。

**这是跨 agent 共享的领域知识库**。

---

## 核心分析维度

## 1. 状态（State）

任何系统都存在状态：
- **显式状态**：变量、字段、标志、数据库记录
- **隐式状态**：顺序、上下文、历史、依赖关系

### 必须回答的问题
- 状态如何创建？
- 状态如何迁移？
- 状态如何失效？
- 状态是否可被跳跃或回滚？
- 状态在不同系统间如何同步？

### 安全研究关注点
- 状态机是否可被绕过？
- 是否存在非法状态转换？
- 状态不一致是否可被利用？

---

## 2. 边界（Boundary）

边界定义"什么可以做，什么不可以"。

### 常见边界类型
- **身份边界**：谁是谁，如何证明
- **权限边界**：能做什么，不能做什么
- **生命周期边界**：资源的创建、使用、销毁
- **组件边界**：模块、服务、系统之间的边界
- **租户边界**：多租户系统间的隔离

### 必须回答的问题
- 边界在哪里？
- 边界如何被检查？
- 边界是否一致（所有路径）？
- 边界是否可被组合绕过？

### 安全研究关注点
- 边界检查是否存在不一致？
- 是否存在跨边界的信任传递？
- 边界是否可被跳跃？

---

## 3. 信任（Trust）

信任是所有漏洞的根源。

### 信任的来源
- **身份信任**：用户、服务、系统是谁
- **能力信任**：用户、服务、系统能做什么
- **数据信任**：输入、配置、环境变量的可信度

### 必须回答的问题
- 系统信任了什么？
- 信任是否可被伪造？
- 信任是否随时间/状态变化？
- 信任是否跨边界传递？

### 安全研究关注点
- 信任假设是否可被操纵？
- 信任传递链是否可被利用？
- 信任失效是否及时？

---

## 4. 输入（Input）

输入不仅是"用户输入"。

### 输入的类型
- **外部数据**：用户输入、API 参数、文件上传
- **内部状态**：数据库、缓存、配置
- **时间与顺序**：请求顺序、并发、时序
- **环境上下文**：HTTP 头、IP 地址、系统时间

### 必须回答的问题
- 输入是否被验证？
- 输入是否影响状态或边界？
- 输入是否可被组合利用？
- 输入是否影响控制流？

### 安全研究关注点
- 输入验证是否完整？
- 输入是否可导致状态异常？
- 输入是否可导致边界绕过？

---

## 5. 时间（Time）

时间是隐形输入，也是最容易被忽视的。

### 时间相关的安全问题
- **顺序依赖**：操作必须按特定顺序执行
- **竞态条件**：多线程/多进程间的时序问题
- **时间窗口**：某些操作必须在特定时间内完成
- **时间侧信道**：响应时间差异泄露信息

### 必须回答的问题
- 顺序是否重要？
- 是否存在时间窗口？
- 是否存在竞态？
- 时间是否可被操纵？

### 安全研究关注点
- TOCTOU（检查与使用之间的时间窗口）
- 并发操作是否原子？
- 超时机制是否可被绕过？

---

## 6. 资源（Resource）

资源包括内存、句柄、连接、权限、文件、网络。

### 资源相关的安全问题
- **资源泄露**：未正确释放资源
- **资源耗尽**：拒绝服务
- **资源滥用**：超出配额使用
- **资源劫持**：其他用户的资源被访问

### 必须回答的问题
- 资源如何分配？
- 资源如何释放？
- 资源是否与状态一致？
- 资源是否有配额限制？

### 安全研究关注点
- 资源生命周期管理是否正确？
- 是否存在资源泄露？
- 是否存在资源耗尽攻击？

---

## 7. 组合（Composition）

漏洞往往来自"合法组件的非法组合"。

### 组合相关的安全问题
- **操作组合**：多个合法操作组合后非法
- **跨边界组合**：跨系统/服务的操作组合
- **时序组合**：按特定顺序执行的操作
- **批量组合**：分批操作绕过单次限制

### 必须回答的问题
- 单独安全的操作，组合后是否安全？
- 是否存在跨边界组合？
- 批量操作是否有额外检查？

### 安全研究关注点
- 业务逻辑是否存在组合漏洞？
- 是否存在原子性破坏？
- 批量操作是否绕过限制？

---

## 常见安全项目领域

### Web 应用安全
**核心关注**：边界、信任、输入
- 认证与授权
- 会话管理
- 输入验证
- API 安全
- 业务逻辑

### 二进制安全
**核心关注**：输入、资源、状态
- 内存破坏
- 逻辑漏洞
- 逆向工程
- 漏洞利用

### 协议安全
**核心关注**：状态、时间、组合
- 状态机
- 时序分析
- 协议 fuzz
- 流量分析

### 云与容器安全
**核心关注**：边界、组合、资源
- 租户隔离
- 权限边界
- 资源配额
- 供应链安全

### 攻防演练系统
**核心关注**：状态、边界、组合
- 演练状态机
- 权限与隔离
- 流程完整性
- 结果可信度

## 跨领域通用模式

### 1. 所有问题都是状态问题
- 漏洞利用 = 改变系统状态到攻击者期望的状态
- 权限提升 = 跨越权限边界
- 拒绝服务 = 系统进入不可用状态

### 2. 所有问题都是边界问题
- 漏洞 = 跨越不应该跨越的边界
- 利用 = 找到边界检查的缺失
- 防御 = 强化边界检查

### 3. 所有问题都是信任问题
- 漏洞 = 信任了不该信任的东西
- 利用 = 伪造信任
- 防御 = 最小化信任

---

## 使用指引

### 对所有 Agent
1. **分析前**：根据项目类型，快速回顾相关维度
2. **分析中**：对照维度检查清单，逐项验证
3. **分析后**：将发现映射到对应维度

### 维度权重参考
- **Web 应用**：边界 > 信任 > 输入 > 状态 > 组合 > 时间 > 资源
- **二进制**：输入 > 资源 > 状态 > 时间 > 组合 > 边界 > 信任
- **协议**：状态 > 时间 > 组合 > 边界 > 输入 > 信任 > 资源
- **演练系统**：状态 > 边界 > 组合 > 信任 > 时间 > 输入 > 资源
